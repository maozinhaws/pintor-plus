## File Descriptions

### Core Application Files
- **app.html**: Main application page containing the entire UI structure
  - Navigation system with hash-based routing
  - All application pages in a single HTML file
  - Service worker registration
  - Google Maps and other external script includes

- **app_script.js**: Main JavaScript application (~4,300 lines)
  - Global state management (S object)
  - Utility functions (formatting, calculations, DOM manipulation)
  - Google Drive integration (GDrive object)
  - Encryption layer (_Vault module)
  - All UI rendering logic
  - Business logic for budgets, clients, suppliers, scheduling
  - PDF generation functionality
  - Service worker communication

- **sw.js**: Service worker for PWA functionality
  - Offline resource caching
  - Background sync for Google Drive
  - Push notifications
  - Background processing

### Landing Page Files
- **index.html**: Public landing page with app showcase
  - Marketing content about the app
  - Feature highlights
  - Screenshots and mockups
  - Privacy and security information

- **privacy-policy.html**: Legal privacy policy document

### Configuration Files
- **vercel.json**: Vercel deployment configuration
  - Security headers (CSP, HSTS, etc.)
  - Content security policy
  - URL rewrites

- **_headers**: Additional HTTP headers configuration
- **_redirects**: URL redirect rules
- **site.webmanifest**: PWA manifest with app metadata
- **package.json**: Minimal dependencies (only lucide-react icons)

### Supporting Scripts
- **apply_fixes.py**: Python utility to fix common code issues
- **check_balance.js** (v1-v3): Different versions of balance checking utilities
- **check_*.js**: Various JavaScript validation utilities
- **check_syntax.py**: Python syntax checker
- **extracted_script.js**: Potentially extracted code from HTML

### Asset Files
- **Favicon files**: Various sizes and formats for different platforms
- **Apple touch icons**: iOS-specific icons
- **Android Chrome icons**: Android-specific icons
- **Image assets**: App screenshots and promotional materials

- **Pages**: pg-home, pg-orc, pg-clientes, pg-fornecedores, pg-agenda, pg-config, pg-login

- **SVG Icons**: Inline SVG icon definitions
- **Templates**: HTML templates for dynamic content

## Technical Patterns

### State Management
- Single global state object (S) with multiple properties
- Direct mutation pattern (not immutable)
- localStorage/sessionStorage for persistence

### UI Architecture
- Single-page application with hash routing
- Declarative rendering functions
- Event delegation for dynamic elements

### Data Flow
- Client-side first with immediate UI updates
- Background sync with Google Drive
- Offline-first approach with automatic reconciliation

### Security
- Client-side encryption for sensitive data
- OAuth 2.0 with Google for authentication
- Content Security Policy for script control
